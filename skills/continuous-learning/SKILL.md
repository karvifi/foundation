# SKILL: Continuous Learning

## Capture Learning

After each project, answer:
```markdown
# Project Learning — [Project Name]

## What Worked Well (keep doing this)
1. [Practice]: [Specific benefit]
2. [Practice]: [Specific benefit]
3. [Practice]: [Specific benefit]

## What Didn't Work (avoid doing this)
1. [Practice]: [Problem it caused]
2. [Practice]: [Problem it caused]

## New Pattern Discovered
[Pattern]: [When to use it] | [How it works]

## Tools That Helped
- [Tool]: [Why useful] | [When to use]

## One Thing to Change Next Time
[Specific practice to change]: [How to change it]

## Metrics
- Actual timeline vs estimate: [comparison]
- Quality: [what percentage passed tests first try]
- Learning: [new things learned]
```

## Knowledge Base

Add learnings to:
```
docs/PATTERNS.md
  - [Pattern 1]: [When/How/Why]
  - [Pattern 2]: [When/How/Why]

docs/PITFALLS.md
  - [Pitfall 1]: [How to avoid]
  - [Pitfall 2]: [How to avoid]

docs/TOOLS_REFERENCE.md
  - [Tool]: [Use case] | [Setup time] | [Cost]
```

## Quarterly Review

Every 3 months:
```markdown
# Q1 2024 Learning Review

## Most useful pattern discovered
[Pattern]: [Used in N projects] | [Save X time]

## Biggest mistake
[Mistake]: [How to prevent]

## Framework improvements
- [Improvement 1]: [Impact]
- [Improvement 2]: [Impact]

## New tools to evaluate
- [Tool]: [For what task]

## Next quarter focus
- [Area to improve]
```

## Quality checks
- [ ] Learning captured after every project
- [ ] Patterns documented with use cases
- [ ] Pitfalls recorded to avoid repeating
- [ ] Knowledge base updated quarterly
- [ ] Metrics tracked (time, quality, learning)
- [ ] Feedback loop: learn → improve → verify
