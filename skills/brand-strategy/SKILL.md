# SKILL: Brand Strategy

## Purpose
A brand is the total experience someone has with you. It lives in their head, not yours.
This skill builds a brand that people remember, trust, and choose — consistently.

## The Brand Architecture

### Layer 1: Brand Foundation (do this before anything visual)

**Brand Purpose** (the WHY — Simon Sinek)
```
WHY:   [why you exist beyond making money — the belief you stand for]
HOW:   [how you deliver on that belief — your approach]
WHAT:  [what you actually sell — said last, not first]

Example:
WHY:   We believe everyone deserves to understand their health data
HOW:   By making complex medical information visually clear
WHAT:  Health data visualization software
```

**Brand Values** (3-5 maximum — more = none)
For each value, define:
- The value: [one word or short phrase]
- What it means in practice: [specific behavior that demonstrates it]
- What it explicitly does NOT mean: [the opposite behavior to avoid]

Example:
- Value: Clarity
  - Means: We explain complex things simply. Plain language always.
  - Not: Dumbed down, oversimplified, or patronizing

**Brand Personality** (the brand as a person)
Rate these axes to find the personality:
```
Serious ←————————→ Playful
Classic ←————————→ Modern
Formal ←————————→ Casual
Expert ←————————→ Accessible
Bold ←————————→ Subtle
Individual ←————————→ Community
```
Then: "If [brand] were a person, they would be like [specific person archetype]. They speak like [example]. They would never [anti-example]."

### Layer 2: Brand Positioning

**Positioning Statement** (internal use — never say this to customers):
```
For [target customer — be specific]
Who [specific problem or desire]
[Brand name] is a [category]
That [primary differentiator]
Unlike [main competitor],
We [key proof of differentiator]
```

**Value Proposition** (external — what you say to customers):
Format: Headline + Subheadline + 3 bullets
```
Headline: [Outcome the customer gets — not what you do]
Subheadline: [How you achieve it — your approach]
• [Specific benefit 1 with proof]
• [Specific benefit 2 with proof]  
• [Specific benefit 3 with proof]
```

### Layer 3: Brand Voice & Tone

**Voice** (consistent — who you are):
Define 4 voice characteristics:
```
Characteristic: [adjective]
Description: [what this means in copy]
Do: [example of right voice]
Don't: [example of wrong voice]
```

Example:
```
Characteristic: Direct
Description: We say what we mean in as few words as possible
Do: "This tool finds bugs before they reach production."
Don't: "This innovative solution leverages AI-powered technology to proactively identify potential issues..."
```

**Tone** (flexible — adjusts by situation):
| Situation | Tone Shift | Example |
|-----------|-----------|---------|
| Error message | Calm, helpful, never blame user | "Looks like something went wrong. Here's how to fix it:" |
| Success message | Warm, celebratory but brief | "Done! Your report is ready." |
| Marketing copy | Energetic, benefit-focused | "Stop chasing bugs. Start shipping features." |
| Documentation | Clear, neutral, task-focused | "To configure the API, navigate to Settings > API Keys" |
| Social media | Conversational, slightly playful | "Turns out fixing bugs is faster when you know where they are 🎯" |

### Layer 4: Brand Story

**The Origin Story** (why this exists):
Follow the Hero's Journey but make the CUSTOMER the hero:
```
1. The world as it was (customer's old reality)
2. The problem that changed everything
3. What the customer tried (that didn't work)
4. The insight that changes everything
5. The transformation (customer's new reality)
6. Your role: you are the GUIDE, not the hero
```

**The Founder Story** (for startups — builds trust):
```
What I personally experienced that made me build this...
What I tried that failed...
What I eventually discovered...
Why I believe this matters for you...
```

### Layer 5: Visual Identity Direction

**Mood Board Requirements** (before logo design):
Collect 20-30 reference images that capture:
- Color feelings (not specific colors yet)
- Typography personality (serif/sans, light/bold, classical/modern)
- Photography style (lifestyle/product/abstract)
- Illustration style (if used)
- Overall aesthetic (minimal/rich/industrial/organic)

**Color Strategy**:
```
Primary color: [1 color] — carries brand recognition
Secondary color: [1-2 colors] — supports primary
Accent color: [1 color] — used sparingly for CTA/highlights
Neutral palette: [white/grays/black] — for backgrounds/text
Semantic colors: success (#22c55e), warning (#f59e0b), error (#ef4444)

For each color define:
- Hex code
- WCAG contrast ratios (4.5:1 for normal text, 3:1 for large text)
- When to use
- When NOT to use
```

**Typography Hierarchy**:
```
Display / H1: [Font name, weight, size range]
Heading / H2-H4: [Font name, weight, size range]  
Body: [Font name, weight, size] — most readable, most used
Caption/Label: [Font name, weight, size]
Mono: [Font name] — for code, data

Pairing rule: Max 2 font families. Often 1 is enough.
```

### Layer 6: Brand Application Rules

**Logo Usage**:
```
Clear space: minimum [X]px on all sides
Minimum size: [X]px wide (below this use icon only)
Acceptable backgrounds: [white/dark/color]
Never: stretch, rotate, recolor, add effects, use on busy backgrounds
Always provide: full color, white, black, icon-only versions
```

**Photography/Illustration Direction**:
```
Subject: [who is shown — real people? abstract? product?]
Style: [photographic treatment — bright/moody/clean/gritty]
Perspective: [eye-level/above/aspirational/candid]
What to avoid: [stock photo clichés specific to your industry]
```

## Brand Audit (for existing brands)

Score each dimension 1-10:
```
□ Consistency: Does the brand look/sound the same everywhere?
□ Differentiation: Is it visually distinct from competitors?
□ Memorability: Would someone recognize it without the logo?
□ Authenticity: Does it match the actual product/company?
□ Scalability: Does it work at any size, any medium?
□ Flexibility: Can it evolve without losing recognition?
```

Score < 6 = needs work. Score < 4 = rebrand required.

## Output
- Brand Foundation Document (purpose, values, personality, positioning)
- Brand Voice Guide (with do/don't examples)
- Visual Direction Brief (mood board guidance, color strategy, type direction)
- Brand Story (origin story + value proposition)
- Brand Application Rules

## Quality checks
- [ ] Purpose statement is NOT "we make [product]" — it's a belief
- [ ] Values have behaviors attached (not just words)
- [ ] Voice has examples — not just descriptions
- [ ] Positioning differentiates from top 3 competitors
- [ ] Visual direction can brief a designer without further explanation
- [ ] Customer is the hero, brand is the guide
