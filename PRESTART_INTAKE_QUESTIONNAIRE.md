# PRESTART INTAKE QUESTIONNAIRE

Do not start implementation until all sections are answered.

## 1. Objective
1. What are you trying to achieve in one sentence?
2. What does success look like in measurable terms?
3. What are the top 3 failure outcomes to avoid?

## 2. Project Type
1. Is this: web app / mobile app / API / automation / AI agent / data platform / other?
2. Is this new, incomplete, or inherited work?
3. What already exists today?

## 3. Users and Scope
1. Who are the primary users?
2. What are the top 5 must-have capabilities?
3. What is explicitly out of scope?

## 4. Technical Preferences
1. Preferred language(s)?
2. Preferred frontend stack (if any)?
3. Preferred backend stack (if any)?
4. Preferred database/storage?
5. Any forbidden technologies?

## 5. Quality and Risk
1. Required reliability/uptime?
2. Required performance targets?
3. Security/compliance requirements (PII, HIPAA, SOC2, etc.)?
4. Acceptable risk level?

## 6. Delivery Constraints
1. Budget constraints?
2. Hosting environment constraints?
3. Timeline and milestones?
4. Team skills and staffing?

## 7. Operations
1. Monitoring/alerting requirements?
2. Incident/rollback expectations?
3. Support and maintenance expectations?

## 8. Acceptance Criteria
1. What tests must pass before release?
2. What user scenarios must be demoable?
3. What documentation is mandatory?

## Exit Criteria
Implementation may start only when all answers are provided or explicitly marked as "auto-decide".
If marked "auto-decide", the foundation must provide ranked recommendations with rationale.
